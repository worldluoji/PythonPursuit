# 大型项目模块化设计
模块应该像洋葱一样分层，依赖方向始终保持一致！
```
大型项目模块化结构示例：
my_project/
├── src/
│   ├── my_project/          # 主包
│   │   ├── __init__.py
│   │   ├── core/           # 核心模块
│   │   │   ├── __init__.py
│   │   │   ├── exceptions.py
│   │   │   ├── config.py
│   │   │   └── base.py
│   │   ├── api/            # API层
│   │   │   ├── __init__.py
│   │   │   ├── routes/
│   │   │   ├── middleware/
│   │   │   └── schemas/
│   │   ├── services/       # 业务逻辑层
│   │   │   ├── __init__.py
│   │   │   ├── user_service.py
│   │   │   ├── order_service.py
│   │   │   └── payment_service.py
│   │   ├── models/         # 数据模型层
│   │   │   ├── __init__.py
│   │   │   ├── user.py
│   │   │   ├── order.py
│   │   │   └── base.py
│   │   ├── repositories/   # 数据访问层
│   │   │   ├── __init__.py
│   │   │   ├── user_repo.py
│   │   │   └── order_repo.py
│   │   └── utils/         # 工具模块
│   │       ├── __init__.py
│   │       ├── validators.py
│   │       ├── helpers.py
│   │       └── decorators.py
│   ├── tests/             # 测试目录
│   │   ├── __init__.py
│   │   ├── conftest.py
│   │   └── test_*.py
│   └── docs/              # 文档
├── pyproject.toml         # 项目配置
├── requirements.txt       # 依赖管理
└── Dockerfile            # 容器化配置
```
