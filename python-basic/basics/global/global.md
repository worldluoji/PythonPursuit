# global
**`global`关键字用于在函数内部声明并修改全局作用域中的变量，使函数能够直接操作全局变量而非创建同名局部变量。**

### 一、核心作用与机制

1. **突破函数作用域限制**  
   在Python中，函数内部默认只能**读取**全局变量，若尝试修改（如赋值操作），Python会**自动创建同名局部变量**，而非修改全局变量。  
   - **示例**：  
     ```python
     count = 0  # 全局变量
     def increment():
         count = 1  # 实际创建局部变量，全局count不变
     increment()
     print(count)  # 输出: 0（未修改成功）
     ```

2. **声明全局变量权限**  
   通过`global`关键字显式声明后，函数内部可**直接修改全局变量**：  
   ```python
   count = 0
   def increment():
       global count  # 声明count为全局变量
       count += 1    # 修改全局count
   increment()
   print(count)  # 输出: 1（修改成功）
   ```

### 二、典型使用场景

1. **修改全局状态**  
   当函数需要更新全局配置或计数器时：  
   ```python
   config = {"mode": "default"}
   def set_mode(new_mode):
       global config
       config["mode"] = new_mode
   ```

2. **多函数共享变量**  
   多个函数需操作同一全局变量（如计数器）：  
   ```python
   total = 0
   def add(num):
       global total
       total += num
   def reset():
       global total
       total = 0
   ```

3. **避免变量命名冲突**  
   当局部变量与全局变量同名时，`global`可明确指定使用全局变量：  
   ```python
   name = "全局变量"
   def test():
       name = "局部变量"  # 默认创建局部变量
       print("局部变量:", name)
   test()  # 输出: 局部变量
   print("全局变量:", name)  # 输出: 全局变量
   ```

### 三、关键注意事项

1. **声明位置要求**  
   - 必须在**修改变量前声明**，但可在函数内任意位置（通常在开头）：  
     ```python
     def func():
         x = 10        # 局部变量
         global y      # 声明y为全局变量
     ```

2. **仅限函数内部使用**  
   - `global`只能在函数内部使用，函数外部使用会报错。

3. **仅针对不可变类型**  
   - 对**不可变类型**（整数、字符串、布尔值、元组）必须使用`global`修改。  
   - **可变类型**（列表、字典）可直接修改，无需`global`。

4. **慎用原则**  
   - 过度使用会降低代码可维护性（变量状态难以追踪）。  
   - **替代方案**：优先通过函数参数传递数据，或使用类的属性管理状态。

### 四、与`nonlocal`关键字的区别

| 特性                | `global`                     | `nonlocal`                   |
|---------------------|-------------------------------|-------------------------------|
| **作用域**          | 全局作用域（模块级别）        | 闭包作用域（外层函数）       |
| **使用场景**       | 修改全局变量                 | 修改嵌套函数外层变量         |
| **声明位置**       | 函数内部                     | 嵌套函数内部                 |
| **示例**           | `global config`              | `nonlocal outer_var`        |

> 💡 **最佳实践**：  
> - **仅在必要时使用**`global`（如全局计数器、配置管理）。  
> - **避免滥用**：全局变量易导致代码耦合，优先考虑函数参数和返回值传递数据。  
> - **命名规范**：全局变量使用`UPPER_CASE`格式（如`MAX_CONNECTIONS`），提高可读性。
